# Лабораторная работа №1

## Вариант 19

### Тема: gRPC-сервис DataProcessor (Client Streaming RPC)

---

## Задание

Реализовать gRPC-сервис DataProcessor с методом:

ProcessDataBatch(stream DataPoint)

Метод должен принимать поток данных (чисел) от клиента и после обработки возвращать один результат  
(Client Streaming RPC).

Клиент вводит несколько значений вручную, отправляет их на сервер, после чего сервер выполняет обработку и возвращает итоговый результат.

---

## Архитектура решения

В работе реализована клиент-серверная архитектура.

flowchart LR

Server -->|Result| Client

Proto --> Generated

Generated --> Server

Компоненты системы

Client (client.py)  
Отправляет поток данных (чисел) на сервер.

Server (server.py)  
Принимает поток данных, обрабатывает их и возвращает результат.

data.proto  
Контракт взаимодействия между клиентом и сервером.  
Описывает сервис и структуры сообщений.

## Описание реализации

В рамках лабораторной работы был создан gRPC-сервис DataProcessor.

Описание сервиса выполнено в файле data.proto.

Сервис содержит метод:

- ProcessDataBatch — принимает поток данных от клиента.

Метод использует ключевое слово stream, что позволяет клиенту отправлять несколько сообщений подряд.

Сервер принимает все значения, выполняет вычисления (сумма и среднее) и возвращает один итоговый результат.

Таким образом реализуется механизм Client Streaming RPC.

## Генерация gRPC-кода

После создания файла data.proto была выполнена команда:

python -m grpc_tools.protoc -I. --python_out=. --grpc_python_out=. data.proto

В результате были автоматически созданы файлы:

- data_pb2.py  
- data_pb2_grpc.py  

Эти файлы используются для взаимодействия клиента и сервера через gRPC.

---

## Реализация сервера

Серверная часть реализована в файле server.py.

Создан класс DataProcessor, который наследуется от DataProcessorServicer.

В данном классе реализован метод:

ProcessDataBatch — принимает поток сообщений DataPoint.

Сервер последовательно получает значения от клиента, сохраняет их, затем выполняет вычисление суммы и среднего значения.

После обработки сервер возвращает клиенту один итоговый результат.

Сервер запускается на порту:

50051

и ожидает подключения клиентов.

## Реализация клиента

Клиентская часть реализована в файле client.py.

Подключение к серверу выполняется через:

grpc.insecure_channel('localhost:50051')

Клиент вручную вводит числа, которые отправляются на сервер как поток сообщений DataPoint.

Ввод продолжается до тех пор, пока пользователь не введет "stop".

После завершения отправки сервер возвращает результат обработки данных.
